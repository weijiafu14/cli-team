# 自媒体素材采集库方案建议书

## 1. 核心目标

解决自媒体创作者（如“公开构建”型创作者）在工作过程中，因专注而无法手动保存素材，导致后续创作缺乏视觉资料的问题。

## 2. 核心技术路径（已验证）

### 2.1 目标导向式采集 (Goal-Driven Capture)

- **原理**：用户设置一个具体工作目标（如：“开发登录界面”），Agent 自动创建一个以此命名的素材夹。
- **优势**：素材从产生的一刻起就是被组织好的，无需事后手动分类。

### 2.2 视觉去重 (dHash 算法)

- **原理**：每帧截图通过 dHash 算法生成指纹。如果当前画面与前一帧相似度过高（Hamming 距离 <= 1），则不保存。
- **优势**：极大地节省了存储空间，且素材库中只保留有意义的变化，方便后续剪辑。

### 2.3 静默后台采集 (Main Process Service)

- **原理**：集成到 AionUi 主进程中，使用 Electron 的 `desktopCapturer` 接口。
- **优势**：不占用渲染进程资源，不干扰用户正常工作和聊天。

## 3. 自动化整理与总结方案

### 3.1 关键帧提取 (Key-Frame Extraction)

- **方案**：会话结束后，自动提取开始、中间、结束状态的关键帧。
- **应用**：直接用于生成“工作流总结”这类极具说服力的自媒体素材。

### 3.2 AI 文案助手

- **方案**：Agent 分析被捕获的截图，结合用户最初设定的工作目标，自动生成对应平台的文案（如：小红书的“工作日常”、X 的“进度更新”）。

## 4. 后续规划

1. **多屏幕支持**：增加对副屏或指定窗口的监听采集。
2. **敏感信息自动遮蔽**：利用本地 Vision 模型自动识别并模糊化截图中的密码、秘钥等敏感区域。
3. **一键转视频**：将采集到的有序截图集自动合成时间流转（Time-lapse）短视频。
